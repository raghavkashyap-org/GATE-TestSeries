Q8 MCQ Not checked
Technical • COMPUTER SCIENCE • +2 / -0.66
Not attempted

Consider the following problems:
P1 : Starvation
P2 : Deadlock
P3 : Inconsistent data
P4 : Priority inversion
Which of the above problem may occur due to the use of locking resources?

Correct answer: D
Solution

(d)
• Starvation: Process P1 wants resource R1, which is locked by process P2, when process P2 completed another process P3 locks resource R1 before process P1. It will happen for indefinite time, so process P1 is in starvation.
• Inconsistent data: This problem occur when one process fail in between and another dependent process read update value of failed process.
• Deadlock: When two process both want two resources to complete but currently lock oneone resources. This will create deadlock since both are waiting for resources to be free.

Consider the following statements:
S1 : Each thread can be scheduled separately.
S2 : Switching between threads is faster.
S3 : Thread frame same process can be run simultaneously on different processors.
S4 : Blocking threads don’t stop the entire process.
Which of the above statement is an advantage of Kernel level threads over user level.

Correct answer: D
Solution

(d)

S2 : Kernel level thread have more context switching than user level thread. So, switching between Kernel level thread is slower.
S3 : Kernel level threads are independent so can be run simultaneously on different process.
S4 : Kernel level threads are designed as independent. So, blocking one threads does not entire process, which is not the case with user level thread.

Q16 MSQ Not checked
Technical • COMPUTER SCIENCE • +1 / -0
Not attempted

Which of the following should be allowed only in Kernel mode?

Correct answer: A, B, D
Solution

(a, b, d)
(a) Change mapping from virtual to physical address: It will be done in Kernel mode only because mapping is done with the help of page table, which are managed by operating system.
(b) Disabling all the interrupts: It required to be operated/handled in Kernel mode since it will effect the function of devices.
(c) Reading time of clock: No need to handle request in Kernel mode.
(d) Mask and unmask interrupt: It will effect the functioning of input/output devices. Hence required to be operated in Kernel mode.
