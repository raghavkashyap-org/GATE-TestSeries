Consider the following SQL statements:

 
Which of the following SQL statements will not generate any error message?
(i) SELECT DEPT, COMMISSION FROM EMP WHERE SALARY > 7000 GROUP BY DEPT
HAVING COMMISSION > 200;
(ii) SELECT EMPNAME, SALARY FROM EMP WHERE COMMISSION = NULL AND DEPT =
‘IT’;
(iii) SELECT * FROM EMP WHERE SALARY BETWEEN 6000 AND 8000 GROUP BY DEPT ORDER
BY EMPID;
(iv)SELECT EMPNAME, DEPT FROM EMP WHERE COMMISSION IS NOT NULL AND SALARY
> 6000;
Correct answer: D, A, B, C
Solution

(d)
(a) This query is incorrect. HAVING is used to filter after a GROUP BY operation and works
with aggregated function.
(b) This query is incorrect. You cannot use = to compare with NULL. The correct way is to use IS
NULL for such comparisons (i.e., COMMISSION IS NULL).
(c) This query is incorrect. There is a syntax error in the SELECT clause (* is invalid). Additionally,
if you use GROUP BY, the columns in the SELECT statement must be part of the GROUP BY
clause).
(d) This query is valid. It properly checks for non-NULL values in the COMMISSION column and
ensures SALARY is greater than 6000.

Q9 MCQ Not checked
Technical • Data Science & Artificial Intelligence • +1 / -0.33
Not attempted

Tables: A(X, Y), B(Y, Z)
• A has 50 rows.
• B has 10 distinct values of Y, each appearing 5 times.
• Natural join: produces 100 rows.
If one row is picked at random from the join, what’s the probability that its Y value appears more
than once in the joined result?

Correct answer: A
Solution

(a)
A has 50 rows → so 50 values of Y.
B has 10 distinct Y values, each appearing 5 times.
Natural join happens on Y.
For join output to be 100 rows, some Y values from A must match those in B. Since each Y in B has
5 rows, any matching Y from A will be joined with all 5 → that Y will appear 5 times in the result.
So, each Y in the join result appears more than once, hence probability = 1.
